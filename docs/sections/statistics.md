# Статистика

Данный раздел предназначен для анализа производительности ваших PWA-приложений. Здесь можно отслеживать ключевые показатели воронки, применять фильтры, смотреть динамику на графике и формировать отчет с возможностью экспорта в CSV формате.

## Верхний блок метрик {#indicators}

В верхней части раздела отображаются основные агрегированные показатели по выбранному периоду и с учетом примененных фильтров:

- **Клики** — количество кликов по ссылке.
- **Контент** — количество пользователей, которые перешли с прелендинга на оффер.
- **Установки** — количество установок PWA.
- **Открытия** — количество открытий приложения.
- **Качество** — соотношение пользователей, которые попали на оффер и совершили целевое действие.
- **Конверсии** — количество зафиксированных целевых действий.
- **Лиды** — количество лидов.
- **Продажи** — количество продаж.
- **Расходы** — расходы по выбранным данным.
- **Доход** — доход по выбранным данным.
- **Прибыль** — разница между доходом и расходами.
- **ROI** — показатель возврата инвестиций.

Все значения в этом блоке пересчитываются в зависимости от выбранных параметров отчета.

## Фильтры {#filters}

Ниже блока метрик расположен набор фильтров, которые позволяют сузить данные и сформировать нужный отчет.

### PWA {#pwa}

Фильтр **PWA** используется для выбора конкретного приложения, по которому нужно получить статистику.

### Страна {#countries}

Фильтр **Страна** позволяет ограничить отчет по выбранным GEO.

### ОС {#os}

Фильтр **ОС** позволяет анализировать данные по разным платформам. В списке могут отображаться как мобильные, так и десктопные операционные системы, например:

- Android
- iOS
- Windows
- macOS
- Linux
- Другое

### Браузеры {#browsers}

Фильтр **Браузеры** позволяет отбирать данные по используемым браузерам. В списке доступны, в том числе, следующие значения:

- Google Chrome
- Safari
- Firefox
- Microsoft Edge
- Opera
- Samsung Internet
- Другое

### Пуши {#pushes}

Фильтр **Пуши** применяется ко всему отчету и позволяет работать с данными по выбранному типу push-событий:

- **Отправлены**
- **Доставлены**
- **Кликнуты**
- **Все**

### Часовой пояс {#time}

Фильтр **Часовой пояс** используется для корректного отображения данных во времени. Это особенно полезно при анализе почасовой динамики и сравнении активности по разным регионам.

### Период {#period}

Фильтр **Период** задает временной диапазон, за который будут собраны данные для графика и отчета.

### Разбивка {#breakdown}

Параметр **Разбивка** используется для группировки данных. Доступны следующие варианты:

- **Дата**
- **Месяц**
- **Страна**
- **ОС**
- **Браузер**
- **Язык**

Разбивка обновляет график в реальном времени. Для применения выбранной разбивки и остальных фильтров к таблице отчета необходимо нажать кнопку **Отчет**.

### Очистить {#clear}

Кнопка **Очистить** сбрасывает выбранные параметры фильтрации.

### Отчёт {#report}

Кнопка **Отчет** применяет выбранные фильтры к таблице ниже графика и формирует отчет, который затем можно скачать в CSV.

Если не нажать кнопку **Отчет**, таблица может оставаться пустой, даже если график уже обновился.

## График {#chart}

Под фильтрами расположен график, который показывает динамику основных показателей за выбранный период.

На графике можно включать и отключать отображение следующих серий:

- **Уники**
- **Установки**
- **Открытия**
- **Лиды**
- **Продажи**

При наведении на точку графика отображаются значения показателей для конкретного времени или выбранной группы данных.

### Основные различия между частью метрик {#difference}

- **Визиты** — все посещения.
- **Уники** — только уникальные посещения.
- **Клики** — клик по ссылке, который не обязательно означает переход на оффер.
- **Контент** — переход пользователя с прелендинга на оффер.
- **Конверсии** — фиксируются, когда пользователь совершает целевое действие.

## Экспорт CSV {#csv_export}

Под таблицей отчета доступна кнопка **Скачать CSV**.

Экспорт позволяет сохранить сформированный отчет в CSV-файл для дальнейшего анализа, передачи команде или загрузки в сторонние инструменты.
