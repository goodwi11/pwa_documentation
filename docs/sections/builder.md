# Конструктор

Конструктор — это основной инструмент для создания и редактирования PWA-приложений. Он состоит из пяти вкладок, которые проходятся последовательно:

1. Основное
2. Домен
3. Контент
4. Пуши
5. Клоакинг

Переход между шагами осуществляется через меню/вкладки внутри конструктора.

## Основное {#main}

На данном этапе настраиваются базовые параметры приложения.

**Название**
Введите название вашего PWA. Это имя будет отображаться на главном экране устройства пользователя и в самом приложении.

**Язык**
Выберите язык интерфейса приложения из выпадающего списка (например, English, Русский).

**Ссылка оффера**
Укажите URL вашего оффера — это адрес, на который будут попадать пользователи из PWA. Вводите полную ссылку с https://.

**Способ создания**

Доступны два варианта:

**Создать из приложения** — парсинг данных из Google Play. Выберите этот вариант, если хотите создать PWA на основе существующего приложения из стора. Система автоматически подтянет:

   - Название и иконку
   - Скриншоты
   - Описание
   - Рейтинг и количество скачиваний
   - Информацию о разработчике

**Создать из своего дизайна** — ручная настройка. Выберите, если хотите загрузить собственные креативы и полностью кастомизировать приложение.

**Ссылка на приложение** (при выборе парсинга)
Вставьте полную ссылку на приложение из Google Play в формате:

`https://play.google.com/store/apps/details?id=com.example.app`

## Домен {#domain}

Для работы PWA необходимо привязать собственный домен. Привязка осуществляется через NS-серверы Cloudflare.

**Шаг 1: Привяжите домен**

В поле **«Домен»** введите ваше доменное имя без протокола и www: `example.com`

Нажмите кнопку **«Получить NS записи»** — система сгенерирует уникальные NS-серверы для вашего домена.

**Шаг 2: Укажите NS-серверы**

Вы получите два NS-сервера вида:

- `NS1: example.ns.cloudflare.com`<br>
- `NS2: example.ns.cloudflare.com`

Перейдите в панель управления вашего регистратора домена и замените текущие NS-серверы на указанные.

**Шаг 3: Проверьте привязку**

!!! important "Важно" 
     Изменения DNS могут занять от нескольких минут до 48 часов. Это зависит от вашего регистратора и текущего TTL записей.

После того как DNS-записи обновятся, нажмите кнопку **«Проверить привязку домена»**. Если всё настроено корректно, вы сможете перейти к следующему шагу.

**Типичные ошибки:**
- Указан домен с www или протоколом — вводите только `domain.com`
- NS-серверы ещё не применились — подождите и повторите проверку
- У регистратора не сохранились изменения — проверьте настройки DNS

## Контент {#content}

На данном этапе настраивается весь контент, который увидят пользователи в вашем PWA. Справа отображается превью приложения в реальном времени.

### Шапка приложения

| Поле | Описание |
|------|----------|
| Иконка | Загрузите иконку приложения (*рекомендуется 512x512 px*) |
| Название | Название приложения в шапке |
| Автор | Имя разработчика/издателя |
| Рейтинг | Средняя оценка (*например, 4.7*) |
| Размер | Размер приложения в МБ |
| Возраст | Возрастное ограничение (*0+, 12+, 18+ и т.д.*) |
| Версия | Номер версии (*например, 1.0.0*) |
| Скачиваний | Количество установок (*например, 10000*) |
| Отзывов | Количество отзывов |

### Скриншоты

Загрузите скриншоты приложения — они будут отображаться в карусели на странице установки. 

- Поддерживаемые форматы: JPEG, PNG
- Рекомендуемое соотношение сторон: 9:16 (вертикальные) или 16:9 (горизонтальные)
- Минимум 4 скриншота для корректного отображения

Нажмите **«Добавить скриншоты +»** чтобы загрузить дополнительные изображения.

### Видео

Опционально можно добавить ссылку на YouTube-видео с обзором или трейлером приложения. Вставьте полную ссылку в формате:
```
https://www.youtube.com/watch?v=XXXXXXXXXXX
```

### Описание и тэги

**Описание**
Текстовое описание приложения. Поддерживает несколько абзацев. Опишите функционал и преимущества — это влияет на конверсию в установку.

**Тэги**
Ключевые слова через запятую, например:
```
game, football, basketball
```

Тэги отображаются как метки под описанием и помогают пользователю понять тематику приложения.

### Оценки и рейтинг

Настройте распределение оценок по звёздам. Используйте слайдеры для указания количества отзывов с каждой оценкой (от 1 до 5 звёзд).

Опция **«Связать оценки с количеством отзывов»** автоматически пересчитает распределение, чтобы сумма соответствовала указанному общему количеству отзывов.

### Отзывы и ответы

Создайте реалистичные отзывы для повышения доверия:

| Поле | Описание |
|------|----------|
| Фото | Аватар автора отзыва |
| Автор | Имя пользователя |
| Рейтинг | Оценка от 1 до 5 звёзд |
| Отзыв | Текст отзыва пользователя |
| Ответ | Ответ разработчика (опционально) |

Нажмите **«Добавить отзыв +»** для добавления нескольких отзывов.

### Контакты разработчика

Укажите контактные данные для раздела «О разработчике»:
- **Адрес** — юридический или физический адрес
- **Электронная почта** — email для связи

После заполнения всех полей нажмите **«Сохранить»** для сохранения контента.


## Пуши {#push}

На данном этапе вы выбираете, какие push-уведомления будут привязаны к создаваемому PWA.

!!! warning "Внимание" 
     Перед использованием этой вкладки необходимо создать пуш-уведомления в разделе **«Пуши»** бокового меню. При первом создании PWA список будет пустым.

**Как это работает:**

1. Сначала создайте нужные пуш-уведомления в разделе «Пуши»
2. Затем при создании или редактировании PWA выберите нужные пуши из списка
3. Выбранные пуши будут отправляться пользователям этого конкретного приложения

**Добавление пушей:**

- Нажмите **«+ Добавить пуш»** для добавления слота
- Выберите нужный пуш из выпадающего списка (например «Push 1», «Push 2»)
- Добавьте несколько пушей при необходимости
- Для удаления нажмите иконку корзины рядом с пушем

Справа отображается превью — так пуш будет выглядеть на устройстве пользователя.

## Клоакинг {#cloaking}

Клоакинг — это система фильтрации трафика, которая позволяет показывать разный контент разным пользователям. Модераторам рекламных сетей показывается вайт, а целевой аудитории — ваше PWA.

### Настройки параметров фильтрации

**Рекламная сеть**

Выберите источник трафика, под который нужно оптимизировать фильтрацию:
- Facebook Ads
- Google Ads
- Taboola
- Propeller

Каждая сеть имеет свои особенности проверки — система учтёт это при фильтрации.

**Режим клоакинга**

Выберите режим работы фильтра:

| Режим | Описание |
|-------|----------|
| Гибкий режим | Оптимальный баланс между проходимостью и защитой. Рекомендуется для большинства случаев |
| Жёсткий режим | Максимальная фильтрация. Используйте при повышенном внимании со стороны модерации |
| Ручной режим | Полный контроль над всеми параметрами фильтрации |

**Страны**

Укажите географию вашего трафика:
- **Разрешить** — только выбранные страны увидят PWA
- **Запретить** — выбранные страны будут видеть вайт

Доступные гео: Россия, Украина, Беларусь, Казахстан, Германия, США и другие.

!!! note "Примечание" 
     Пустое поле означает «разрешить все».

**Операционные системы**

Фильтрация по ОС устройства:
- Windows
- Android  
- iOS
- macOS
- Linux

Работает аналогично странам — можно разрешить или запретить.

**Браузеры**

Фильтрация по браузеру пользователя:
- Chrome
- Safari
- Opera
- Firefox
- Edge

### Настройка вайта

**Способ генерации вайта**

Вайт — это страница с нейтральным контентом, которая показывается модераторам вместо вашего PWA.

Доступны два способа:
- **Из приложения** — система автоматически сгенерирует вайт на основе выбранного приложения из Google Play
- **Из своего дизайна** — выберите настройки вайта самостоятельно

После настройки всех параметров нажмите **«Завершить создание»** — ваше PWA будет создано и станет доступно по указанному домену.
