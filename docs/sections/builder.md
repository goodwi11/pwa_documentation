Конструктор
===========

Данный раздел — это пошаговый мастер создания PWA. Он состоит из 5 основных этапов:

1. Основное
2. Домен
3. Контент
4. Пуши
5. Клоакинг

Переход между шагами осуществляется через меню/вкладки внутри конструктора.

Основное
--------

На шаге «Основное» настраиваются базовые параметры приложения.

**Поля:**

- Название приложения — отображаемое название PWA. Пользователь видит его в шапке приложения и при установке на устройство.
- Язык — основной язык интерфейса приложения (например, русский, английский). Влияет на локализацию текста и шаблонов.
- Ссылка оффера — целевая ссылка, на которую должен вести трафик (например, лендинг партнёрской программы).

**Дополнительные действия:**

- Создать из приложения — создаёт новое приложение на основе уже существующего (клонирование структуры и/или контента).
- Создать из своего дизайна — позволяет использовать собственный дизайн (кастомную верстку или шаблон), вместо стандартных шаблонов конструктора.


Домен
-----

Шаг «Домен» отвечает за привязку домена к приложению.

Этапы:

1. Привяжите домен:

   Укажите доменное имя, которое будет использоваться для приложения (например, app.example.com).

2. Укажите NS-серверы:

   В интерфейсе доменного регистратора необходимо прописать NS-серверы, предоставленные сервисом, чтобы домен корректно указывал на хостинг PWA.

3. Проверка привязки:

   Сервис автоматически проверяет, правильно ли настроены DNS-записи.

   - Если всё корректно — домен получает статус «Подключен».
   - Если есть ошибки — отображается уведомление (например, «NS не обновлены» или «DNS ещё не применились»).


Контент
-------

Шаг «Контент» позволяет полностью настроить внешний вид карточки приложения и его содержимое.

**Шапка приложения:**

- Название — отображаемое название приложения.
- Автор — имя или название разработчика/компании.
- Размер (МБ) — условный размер приложения.
- Возраст — возрастной рейтинг (например, 3+, 12+).
- Скачиваний — количество установок/скачиваний (можно указать любое логичное значение).

**Рейтинг:**

- Общий рейтинг приложения (1–5), который видят пользователи.
- Может рассчитываться автоматически на основе блока «Оценки и рейтинг».

**Скриншоты / фото:**

- Можно загрузить несколько скриншотов или фотографий приложения.
- Кнопка «Загрузить» — добавляет изображение с устройства.
- Есть кнопка, которая увеличивает количество слотов для скриншотов на 2 (каждое нажатие добавляет ещё две позиции для загрузки).

**Видео:**

- Необязательный блок.
- Можно добавить ссылку на видео (например, с YouTube) в качестве промо-ролика приложения.

**Описание и тэги:**

- Описание — текстовое описание приложения (функции, преимущества, сценарии использования).
- Тэги — ключевые слова, вводятся через запятую (например: игра, квест, offline, головоломка). Теги помогают в фильтрации и внутреннем поиске.

**Оценки и рейтинг:**

- Для каждой оценки от 1 до 5 есть ползунок/поле с количеством оценок.
- Вы можете задать любое количество оценок для каждого уровня (например: 5★ — 1200, 4★ — 300 и т.д.).
- На основе этих значений рассчитывается общий рейтинг приложения, который отображается пользователям.

**Отзывы и ответы:**

- Для каждого отзыва задаются:

  - Автор — ФИО или никнейм пользователя, при необходимости можно указать фото.
  - Оценка — рейтинг от 1 до 5.
  - Текст отзыва — комментарий пользователя.
  - Ответ разработчика — официальный ответ от автора/разработчика приложения.

- Кнопка «Добавить отзыв» позволяет создать несколько отзывов подряд.

**Контакт разработчика:**

- Адрес — юридический или почтовый адрес разработчика.
- Электронная почта — контактный email для связи (поддержка, обратная связь).


Пуши
----

Шаг «Пуши» отвечает за привязку к приложению push-уведомлений.

**Добавленные пуши:**

- Отображается список ранее созданных пушей (шаблонов или кампаний).
- Можно выбрать один из уже созданных пушей и привязать его к текущему приложению.

**Создание нового пуша:**

- Если нет ни одного пуша, сначала нужно создать пуш (через раздел «Пуши» или кнопку в конструкторе, если предусмотрено), а затем выбрать его из списка.
- После привязки пуш-уведомления можно использовать для вовлечения и удержания пользователей приложения.

Клоакинг
--------

Шаг «Клоакинг» позволяет настроить логику показа приложения/вайтов в зависимости от источника трафика и параметров пользователя.

**Поля и настройки:**

- Рекламная сеть — выбор рекламной сети, с которой работает кампания (например, FB, Google, TikTok и т.п.).
- Режим клоакинга — режим работы (показ вайта для модерации, редирект на оффер и т.д., в зависимости от логики сервиса).
- Разрешенные страны (необязательно) — список стран, из которых трафик считается валидным.
- Разрешенные ОС (необязательно) — операционные системы, для которых должна срабатывать целевая логика (Android, iOS, Windows и т.д.).
- Разрешенные браузеры (необязательно) — список браузеров (Chrome, Safari, Firefox и др.).
- Разрешенные языки (необязательно) — локализация/языковые настройки пользователя (например, ru-RU, en-US).


**Способ генерации вайта:**

- Из приложения — вайт генерируется автоматически на основе текущего приложения (его контента и оформления).
- Из своего дизайна — можно использовать собственный дизайн вайта (загруженный шаблон или кастомный макет).
